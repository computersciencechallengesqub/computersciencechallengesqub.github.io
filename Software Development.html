Software Development

Commercial software has always been difficult to sell. There are many forces making it difficult to commercialise software products including: widespread piracy, predatory plagiarising rivals, powerful free open source alternatives, companies like Google offering polished free alternatives in exchange for user data. Similarly, the distrust from customers has grown, driven by virus/adware, spyware, predatory monetisation, lock-in, security breaches, usability failures and more. To create a viable software-based business requires addressing all of these issues and more. For the lucky few, particularly those that create stable, subscription based software (Adobe, Microsoft, Oracle, Blizzard), enormous financial success can be obtained. This success is not exclusive to large firms and many software businesses start by providing niche products to companies and communities. Rent-seeking monopolies are often the most profitable business models but they are generally bad for customers. Organisations seek to lock-in users to their applications, operating systems, development platforms, languages etc. so that they can minimise competition, maximise the value of their own knowledge/products and increase incentives for users to pay for features and services. This has lead to an endless churn of developers abandoning bloated unstable frameworks whose lockin undermines the use of modern functionality/techniques. Developers switch to leaner alternatives, directly designed for new technologies, only to see the new alternatives become bloated and unstable in turn.

This poses a challenge for developers wishing to create innovative products of lasting value. Thankfully, because of Turing Completeness, the underlying datastructures and algorithms that make applications function are not fundamentally dependent on any platform. Over the years those that have experienced the limitations of large frameworks have developed lean cross-platform alternatives and minimal libraries to enable powerful, high performance software to be rapidly developed and easily ported to many systems and technologies. As operating systems have become bloated and unstable, more and more application functionality is moving to cloud services that run on rented servers running minimal operating systems and virtualisation frameworks such as Docker. Many applications are now partially or fully cloud based and these carry many advantages in terms of piracy protection, patching and user monitoring/feedback. Modern technology companies tend to create products that span many platforms, particularly web and mobile and have relatively seamless integration. In addition, UI frameworks have tended towards excessive complexity and limited customisation, but with modern html, css and javascript an attractive user interface can be rapidly created and adapted. As with software frameworks, web frameworks (and HTML CSS and javascript) tend towards bloated, hard to debug unstable platforms with lockin. However, alternatives exist and new technologies, such as compiling other languages to javascript (Emscripten) enables high performance innovative web applications with advanced functionality. For example, 3d graphics, VR, AR, deep learning, etc.

PC, mobile and web technology, can be developed within many of the challenge areas. However, this area is focused on the creation of libraries, tools and example code to enable others to easily create high quality applications with advanced functionality. It is less about making a specific product and more about making it easier for others to make innovative software.

Innovation areas for application development:

A. Cross platform (and platform version/capability), high performance, high functionality software

i) Cross language LLVM is the common backend for many programming language compilers. More recently LLVM has been used to convert computer code into optimised javascript to create powerful client side web applications. Create a system for producing source code in different languages from LLVM‚Äôs internal abstract syntax tree representation (known as IR). Use keyword functions to generate language specific functionality such as windowing systems/ file access etc. (https://stackoverflow.com/questions/19385878/converting-llvm-ir-to-other-language-code). The goal of such a system is to enable the creation of debuggable, understandable libraries that can be used with all popular languages (and to assist with the porting of existing code between languages).

ii) Native high-quality experiences on each platform Applications that enable the use of features specific to a given platform. For example, always on top windows, dockable/background applications, transparent windows etc. (https://stackoverflow.com/questions/4052940/how-to-make-an-opengl-rendering-context-with-transparent-background). To enable cross platform functionality, people often use scripting or byte code based languages. For high performance applications, such as computer games, it is more common to have a preprocessor create native code and build files for each platform. The preprocessor enables specialist native code to be embedded in the codebase. This also enables the use of platform specific debugging tools. The goal of this project is to make a preprocessor for IOS, Android, Windows and/or Linux that will create compilable projects using popular development tools for each platform e.g. Xcode, Android Studio, Visual Studio, GCC/Make enabling relatively easy native functionality and build tool usage not available on existing frameworks such as Flutter, Java etc.

iii) Easy development on novel platforms Take a new platform and create an automated development tool that can be easily installed to enable developers who are learning to program to be able to create executables that run on both a PC and the new platform. For example Smart TV, Voice assistant, smart watch, VR (Oculus Go/Quest/Vive)

iv) Supports a diverse range of input devices and display sizes/configurations, with abstracted interfaces on each platform Create a minimal example application that demonstrates how common game UI components such as 3D movement (of camera and object), menus and text data entry can be abstracted for cross interface software development. By abstracting the interface, have the same application code be able to use many different input platforms. Gamepads, VR controllers, hand trackers, ipad, ipencil, mobile phone, smart watch, smart TV etc. (design how all the interfaces might work but only implement your favourite that you have time to develop)

v) Multi-screen applications - Interface on one platform that can control software on another Such as the Nvidia shield mobile app and Smart TV. Create a minimal example project that others can build on, that uses network communication between programs installed on mulitple devices such that a developer can easily make use of remote screens and interfaces as if they were all on the same device. For example local and remote openGL windows or local and remote HTML ui components.

vi) Web apps as exes. Use wxwindows, android studio or xcode to create a minimal web browser application (using a webkit based web browser ui element) and a linked virtual machine (java, python or javascript) and an interface between them to enable webserver based web apps to be converted to self contained executables without needing to use a webserver or web browser.

vii) Photoreal virtual world internet interface. Adapt Unreal or Unity as a renderer for internet loaded content with a virtual machine based scripting engine e.g. java or python. So that a virtual world with dynamic content can be created like a web site (3D environment is like the HTML DOM, virtual machine is like javascript, actions in the world trigger new 'pages' to be loaded). In the case of Unreal we have already made progress adapting it for dynamic loading of virtual assets and control of them in the world via a memory mapped file interface.

B. Respectful, convenient, polished, powerful software

i) Minimised time to usable state Create a packaging tool, library and example application that can be used to create applications with a large quantity of code and assets that are downloaded, decompressed and installed (and uninstalled if resources are limited) on demand. Ensure the initial application is small, downloads quickly, is self contained (no installation needed), launches virtually instantly, can be interactively used with a fixed size set of compressed assets and code. Will download extra functionality as needed in a background thread, starting with a minimal compressed version and seamlessly loading more functionality and assets as needed. Gracefully rolls back heavyweight code and assets as not needed. Time permitting enable the application to be downloaded and streamed from a local network server.

ii) Intelligent diagnostics and autoconfiguration of common problems

Create an example application that requires high quality graphics, GPU accelerated computing, access through a firewall or some other form of operating system configuration that is non-trivial for a non-technically skilled user. Create a library or executable to automate the analysis of the host operating system guide and where possible automating the tasks needed of a user in configuring the PC or mobile device for the specialised application. This system should be robust enough to recover from user error. The system should also be data driven and log its operation, taking updates from a remote server so that the installer can be updated as new issues arise. Time permitting the privacy and security issues of this system should also be examined.

iv) Seamless offline and online functionality

Create an example of a simple web application, such as a notepad-like text editor, that others can adapt for their own purposes. The application should operate like google docs offline mode. The web application can work locally in a private way, can connect to a local network server or can work with a cloud server. Changes to files should be synced when connectivity to a server is established. File changes should be saved so that undo is possible and changes on multiple platforms can be merged/rolled back. Like a git repository. If needed, create a minimal local executable webserver, or mobile app to enable the application to run on different platforms without network access.

vii) Glitch free progressive downloading of interface/assets.

Create a development tool and user interface library for simulating and replicating streamed interface experiences on mobile, PC or web. For example, the progressive loading of a large webpage with images. Be able to simulate low and/or unreliable bandwidth. Create a tool to convert an interface, such as an HTML page, into a highly compressed approximate version that has the same layout, colour scheme and interface components so that the page is instantly responsive. Have the library gradually and smoothly transition to the full content as assets become available.

iv) Always responsive, patched, controlled application

Create an example wrapper executable and library for an application that monitors the application‚Äôs interactivity and resource usage and automatically recovers from crashes, memory leaks, and excessive cpu/gpu usage. This wrapper should also log issues and facilitate patching of the application without explicit restarting. Time permitting the library should support sharing state with the wrapper executable (for example through a memory mapped file) so that the restarted application can continue from where it left off in a relatively seamless way.

vii) Easy cloud processing.

Make it easy for developers to send expensive processing to local or cloud servers for high end functionality on low resource platforms. Create a preprocessor for your code so that functions can be labeled (or if you have time automatically identified using performance measurement) and can be modified so that they can send the function parameters to a server and get it to perform the operation and return the result. The preprocessor should modify the application code and also create the server code so that the application can make use of the server if it is available.

Develop a minimal logging system library and example application that gathers diagnostic platform information and records user interactions through the interface. ii) Cross platform testing, performance and security analysis system

Develop a system that uses virtual machine/simulators/real hardware to replicate as diverse range of platforms that an application can be run in. iii) Data driven user interface testing after launch

Create a library and example application where the interface to a program is data driven and can be updated remotely, so that selected users can have experimental variations in their interface and their interactions can be logged to experimentally validate if the user interface has improved.

Create a system for rewarding internal developers and/or trusted users when they identify reproducible issues. Create a minimal library and example application that enables a user to request functionality or report bugs from the software.
