Computing Infrastructure

Almost all businesses have some form of technical support, in many cases via an internal IT department. However, unlike application development, many senior executives see IT as a cost to be minimised rather than a source of profit. As with software frameworks, IT frameworks and hardware providers are focused on creating lock-in and extracting subscription fees. In addition, as IT departments are typically responsible for software purchases, there is a strong incentive for software companies to not undermine/automate professional IT staff. This is possibly one reason why PC networking remains a usability nightmare for non-technical users. Being responsible for security within an organisation and being so critical to its infrastructure, those in IT roles can get a bad reputation (BOFH). In particular, if corporate spending and security policies introduce significant delays, e.g. months to get computer resources for a new project, then they can be seen as a problem. IT departments can be viewed as holding back the ambitions of the organisation. As a result, innovative technology businesses have started to focus more heavily on using cloud computing providers, such as AWS. Software is developed to run entirely on rented cloud computing resources (so called serverless software development). Most recently, high tech firms have started to emphasise NoOps approaches where as many as possible of the IT support roles are automated and the remaining tasks are managed by application development staff. Businesses such as Instagram (sold for a billion dollars with 13 employees) and Netflix (The firm that coined the term NoOps) have demonstrated the success of this approach.

Non-technology businesses without development teams also require computing infrastructure and large firms such as SAP have had success selling a standardised set of enterprise software to help them automate and manage their core operations. Over time, open source alternatives have been developed and we are approaching the point where a complete IT infrastructure for an organisation could be deployed at minimal cost using cloud services, open source software and low-cost computing hardware. Future IT priorities could shift from maintaining legacy custom software and hardware to deploying IoT sensors, building digital doubles of an organisation and working with management and data analysts to optimise and grow business processes via simulations and artificial intelligence.

A. Rapid/reconfigurable computing setup/normalisation

i) Design and implement a network boot hardware platform to easily setup and manage a diverse range of RaspberryPi based computer interfaces and sensors: e.g. point of sale terminals, information screens, PC style terminals for administrative tasks, IoT sensors, conference calls, electronic door locks, CCTV, vending machines etc. Make two examples, a PC alternative and a non-pc version.

ii) Create a custom live usb/network boot version of an operating system to enable the rapid configuring of pc servers, laptops etc. which will automatically connect to the network and provide their resources and/or network login interfaces without any manual configuration. Just plug the usb stick in to the pc, boot from it and its all ready to go. Time permitting create a centrally managed system to patch operating systems and install and update software.

iii) Develop a Network boot or live usb PC operating system that turns the device into a Network Attached Storage device. Have such devices automatically advertise their availability to the network and automate the versioning of files via Git-style repositories and redundant backup. The ultimate goal of such a system is for users to have complete faith in their data storage and to be able to casually grow it without significant effort or skill. Time permitting explore how this system could be seamlessly integrated with cloud storage providers, such as S3 and Google Drive. This system should include an examination of the data protection and privacy issues that may arise from such a system.

iv) Create a tool to automatically setup and manage centrally managed windows 10 Nvidia pcs for running games or custom 3d accelerated graphics applications via gamestream to raspberry pis or Nvidia shields (https://pimylifeup.com/raspberry-pi-nvidia-shield/). Time permitting explore how an application such as Blender could be adapted to run over the gamestream system.

iv) Create software to enable mobile devices and guest laptops to securely interface with a network and to be remotely monitored and controlled by a central security server. Time permitting, explore how the devices‚Äô alerts, displays, speakers, cameras and/or input can be controlled via a central restful interface to enable the rapid development and deployment of distributed communication/information/IoT sensing etc. applications. This project also requires an analysis of the data protection and privacy implications of the technology.

B. Cyber and physical security digital nervous system

i) Use software defined radio monitoring of wifi and/or mobile phone transmissions to detect unauthorised wifi man in the middle attacks or attempts at transmitting data secretly out of an organisation. i.e. wifi transmissions that are from registered locations/devices. Time permitting explore how anomalous transmissions can be detected using machine learning based modelling of normal network behaviour. This project also requires an analysis of the data protection and privacy implications of the technology.

ii) Create a raspberry pi, npcap and wireshark based packet sniffing network monitor that can be attached to wired and wireless networks and passively monitor network communications. Time permitting explore how anomalous transmissions can be detected using machine learning based modelling of normal network behaviour. This project also requires an analysis of the data protection and privacy implications of the technology.

iii) Develop IoT based physical sensors, that monitor: power usage, room occupancy, face biometrics or other properties to identify potential physical attacks on the network or computer devices. This project includes an analysis of how to formalise physical environments and model plausible/possible attacks. Time permitting explore how anomalous behaviour can be detected using machine learning based modelling of normal physical actions. This project also requires an analysis of the data protection and privacy implications of the technology.

iv) Create an addition to an operating system to steganographically (hide data within) label files/communications to automatically track movement of data within and without an organisation. This project also requires an analysis of the data protection and privacy implications of the technology.

C. Automatically configured, self monitoring, redundant networking and storage infrastructure

i) Create a tool to automatically configure commodity wifi hardware via software (e.g. web scraping and updating web interfaces for wifi switches) to facilitate plug and play of large networks. Time permitting create a Raspberry pi, wifi switch monitor that can power on/power cycle/reset the switch if crashed (via a hardware relay or other circuit) and seamlessly reconfigure the switch to create a redundant wifi system that can recover from hardware failure. Time permitting explore how software can be placed on PCs and or raspberry pis to auto configure their networking when powered on, by searching for a trusted controller and following its configuration instructions. This software should change network settings as needed if there are physical faults on the network to minimise network downtime. This project also requires an analysis of the network security implications of the technology

D. Complete digital double simulation of IT infrastructure

i) Develop a system that enables the computing infrastructure of an organisation (both hardware and software) to be simulated virtually, enabling routine aggressive cyber security penetration testing and fault response testing. Time permitting, get Kali linux running on a connected machine and use it to automate penetration testing of the infrastructure. Alternatively, explore how installed applications and network monitoring tools such as Nmap can be used to automatically identify the hardware and software configuration of an organisation‚Äôs infrastructure so that it can be automatically simulated and attacked virtually.

ii) Develop an anonymisation framework for gathering, storing and analysing an encrypted log of all of the operations on a computer system (key logging, webcam monitoring, screen capture, etc.) from which security and usage metrics can be obtained without invading users privacy (possibly using homomorphic encryption techniques)

iii) Create an example of a complete enterprise software system using open source software with simulated human interactions. Ensure the system can be run for real and its interactions logged so that it can be used to create plausible, replayable interactions. Ensure the software can be used to analyse case studies of infrastructure failures, cyber security issues as well as analysing how to make administration more efficient and user friendly. Include some example applications such as: login/account management, email, wikis, network storage e.g. FTP, financial management, scheduling, project planning. The goal is to produce a complex, realistic system that can be fully automatically setup and adapted and is as simple to maintain as possible. Creating the full system is too much for a single project but making a start towards this could be very valuable.

E. Remote software development infrastructure

i) Create an easy to install, remote software development environment so that resource intensive programs can be developed on commodity hardware (For example, https://github.com/cdr/code-server). Time permitting create tools to enable the development environment to easily setup docker installations. The goal being that you can easily run and debug applications on a third machine (ideally including GPU support). In other words, one PC for the interface (e.g. raspberry pi/tablet/phone), one PC for the development environment and one or more PCs for running and debugging applications.

ii) To ensure private communication on monitored/unstrustworthy networks such as free wifi in cafes/trains/hotels etc. Create a Virtual network card device driver that encrypts and hides a PCs network communications. Network packets should be hidden within seemingly normal http communications with one or more trusted servers. Time permitting the system could include 2 factor authentication between your device and the remote network (they validate you and you validate to them). Potentially use public channels such as social media posts if a mobile phone is not available.
