Automated Creativity

In 2018 and 2019 there were a number of significant advances in Artificial Intelligence showing how deep-learning based algorithms and large datasets could be used to create original content at a human or superhuman level. For example, StyleGAN from Nvidia can synthesise photorealistic images of novel human faces at a quality beyond that of even the most talented human artists. Similarly, GPT2 from OpenAI can synthesise interesting written articles on a very large range of topics. StyleGAN inspired technology has already been commercialised in the top selling FaceApp and there are many other projects seeking to build on these and other automated creativity projects.

A specific project in this topic is a combination of a step in the process e.g. A. Formalise and a type of creative product to be created e.g. i) Birthday/Seasonal cards. Time permitting multiple steps can be completed to get closer to a complete automated system for creating a product. However, at least one part of the project should be polished to high standard so that it has lasting value as a component that others can build on.

To enable a computer to creatively produce content it requires the following elements:

A. Formalise how content can be defined in a data driven way.

Typically this format is under-constrained, in other words, the representation can include many forms that wouldn‚Äôt be valid. For example, an image where pixels can take any colour could be used as the representation for a picture of a face. For deep learning applications it is generally necessary to convert a representation into a fixed length array of numbers so that existing algorithms can manipulate it. In some cases where very unconstrained representations lead to poor results it can be more effective to start with very specific examples and then to gradually generalise them. For example in text synthesis similar adjectives and proper nouns are swapped out (templated) to create procedural sentences with similar meanings. This has been used successfully to automatically the production of legal documents.

B. Create a numeric metric for success.

First by gathering a large dataset of examples of the thing you are trying to produce. The higher the quality and the larger the size the better. Ideally these examples need to be transformed into the same formal representation that the system uses to output its results. If this is not possible, features of existing examples can be extracted from the AIs output and existing data samples to provide a judge of whether the results are similar. For example, matching the main colours that have been used in an image to ensure images are colour coordinated.

C. Use existing techniques such as GANs or constraint solving algorithms to learn how to synthesise examples that are similar to the datasets.

Use existing techniques to penalise answers that are very similar to existing training data to encourage the system to create novel examples. If existing techniques are ineffective then novel machine learning approaches can be explored. This can lead to valuable contributions to machine learning.

D. Add tags or other properties to training samples to create a parametrisable synthesis algorithm

For example by labelling the age of peoples faces, a system can be produced where that property is adjusted, enabling the creation of the FaceApp tool for aging peoples faces.

E. Interactive editing.

While fully automated examples are exciting, commercial products are still viable where these tools are used to create suggestions and help refine products made in collaboration with a human designer.
